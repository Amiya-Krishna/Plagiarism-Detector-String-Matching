⚙️ Algorithm Details

Knuth-Morris-Pratt (KMP)

KMP builds an LPS (Longest Proper Prefix which is also Suffix) array for the pattern, enabling it to skip characters that we know won't match — avoiding redundant comparisons. When a mismatch occurs at position j, instead of starting over, we consult the LPS array to jump back only as far as needed.

Time: O(n + m) Space: O(m) Best for: exact substring search

Rabin-Karp Rolling Hash

Rabin-Karp hashes the pattern, then slides a window of the same length across the text. The rolling hash is recomputed in O(1) per step by dropping the leftmost character and adding the new rightmost character. Hash collisions trigger a full character comparison to confirm.

Time: O(n + m) avg Space: O(1) Best for: multi-pattern search

Jaccard Similarity (Shingling)

Documents are converted to sets of overlapping k-character substrings (shingles). Jaccard similarity is then: |A ∩ B| / |A ∪ B|. A value near 1 means the documents share nearly all their character-level patterns, indicating near-duplication even after minor edits.

Formula: |A ∩ B| / |A ∪ B| Shingle k: 5 chars

Cosine Similarity (TF Vectors)

Both documents are represented as term-frequency vectors in word-space. Cosine similarity measures the angle between these vectors: dot(A,B) / (||A|| × ||B||). It is length-invariant — a short paragraph copied into a long essay still scores high on the relevant dimensions.

Formula: dot(A,B)/(‖A‖·‖B‖) Best for: vocabulary distribution
